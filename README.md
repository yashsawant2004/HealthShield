# Health Insurance Policy Recommender

A live recommendation system: as the user fills in personal info, health
info, and coverage requirements, the top 5 best-fit policies update
automatically (no "submit" button needed).

## Project Structure

```
health_insurance_recommender/
├── backend/
│   ├── app.py              # Flask API (serves frontend + /api endpoints)
│   ├── recommender.py      # Scoring engine (the recommendation logic)
│   ├── brochure.py         # On-demand PDF brochure generator (reportlab)
│   ├── data_processor.py   # One-time ETL: messy xlsx -> clean JSON
│   ├── brochures/          # Generated/cached brochure PDFs (auto-created)
│   └── master_policies.xlsx
├── data/
│   └── policies_clean.json # Generated by data_processor.py
├── frontend/
│   ├── index.html          # Form + 3-tab results UI
│   ├── style.css
│   └── script.js           # Calls the API on every input change
└── requirements.txt
```

## How it works

1. **data_processor.py** (run once, or whenever the source spreadsheet
   changes): the raw sheet has very inconsistent free-text formatting
   (e.g. sum insured written as "₹3 lac – ₹50 lac" or "25 Lakh / 50 Lakh
   / 100 Lakh"). This script uses regex-based best-effort parsing to
   extract structured fields: numeric sum-insured range, min/max age,
   yes/no coverage flags (maternity, OPD, critical illness, etc.),
   rating, premium (where disclosed), and category flags like
   family-floater / senior-citizen / top-up. Output -> `data/policies_clean.json`.

2. **recommender.py**: a transparent, weighted rule-based scorer (not a
   black-box ML model — appropriate given ~100 policies and no historical
   click/conversion data to train on).
   - Hard filters first eliminate clearly ineligible policies (age out of
     range, sum insured far too low).
   - Remaining policies get points for: sum-insured fit, budget fit,
     family-floater match, pre-existing-disease waiting period, existing
     medical condition coverage, BMI-risk-friendly terms, each selected
     requirement (maternity/OPD/critical illness/etc.), a free-text
     **"Others"** requirement (keyword-matched against each policy's
     description), senior-citizen friendliness, policy rating, and
     hospital network size.
   - Each policy gets a 0–100 `match_score` plus a short list of
     human-readable `match_reasons` shown on the card.
   - `recommend_topup()` runs the same scorer but restricted to
     top-up/super-top-up policies — surfaced in the **"Top-Up for
     Renewal"** tab for users who already hold a base policy.
   - `top_overall()` returns a profile-independent **Top 10** list
     ranked by disclosed rating, sum insured ceiling, and hospital
     network size — shown in the **"Top 10 Best Overall"** tab.

3. **brochure.py**: the source spreadsheet doesn't ship actual insurer
   PDF brochures, so this module generates a clean one-page PDF summary
   per policy on first request (cached in `backend/brochures/`) using
   `reportlab` — key facts table, coverage highlights, and a disclaimer
   pointing to the insurer's official document for binding terms.

4. **app.py**: a thin Flask layer.
   - `POST /api/recommend` — personalized top-N (default 5) matches.
   - `POST /api/topup` — top-up/super-top-up matches (for the renewal tab).
   - `GET /api/top10` — profile-independent best-of-catalog list.
   - `GET /api/brochure/<policy_id>` — streams the generated PDF brochure.
   - `GET /api/meta` — catalog stats.

5. **frontend**: a single form, now with three tabs:
   - **Personalized Matches** — live top 5 for the entered profile.
   - **Top-Up for Renewal** — live top 5 top-up plans; check "I already
     have a health policy" to tailor it with your current insurer/SI.
   - **Top 10 Best Overall** — static best-of-catalog list, loaded once.

   The form now covers: age, gender, marital status, occupation, city
   tier, family floater + size, height/weight (auto-computed BMI),
   pre-existing disease toggle, a multi-select list of existing medical
   conditions, smoker/alcohol toggles, sum insured slider, budget slider,
   room-type preference, premium frequency preference, renewal toggle
   (current insurer + current SI), a requirements checklist, and an
   **"Others"** free-text box for anything not covered by the checklist.
   Every input has a `change`/`input` listener that triggers a
   **debounced (300ms) fetch**, so both the personalized and top-up tabs
   re-rank live as you adjust any field — no page reload, no submit
   button. Each result card has a **"View Brochure (PDF)"** link that
   opens the generated brochure in a new tab.

## Running it

```bash
cd health_insurance_recommender/backend
pip install -r ../requirements.txt

# 1. Generate the clean dataset (only needed once, or after updating the xlsx)
python3 data_processor.py

# 2. Start the server
python3 app.py
```

Then open **http://127.0.0.1:5000** in your browser.

## Notes / data caveats

- The source spreadsheet rarely discloses exact premiums (~92% of rows say
  "refer rate chart" instead of a number), so budget is treated as a soft
  preference, not a hard cutoff, whenever a premium figure isn't available.
- `claim_settlement_ratio` is almost never disclosed in the sheet either, so
  it isn't used as a scoring factor; `policy_rating` is used instead.
- Parsing is heuristic/regex-based since the source is free text. For a
  production system, the data team would ideally supply a structured
  spreadsheet (separate numeric columns for min/max sum insured, min/max
  age, premium, etc.) to make this fully reliable and to enable a hard
  budget filter.

## Extending this

- Swap `recommender.py`'s rule-based scorer for a learned model (e.g.
  logistic regression / gradient boosting on user clicks → purchases)
  once you have interaction data, while keeping the same API contract.
- Add authentication + saved profiles so returning users don't have to
  re-enter their info.
- Add a comparison view (select 2-3 cards → side-by-side table).
